En aplicaciones web modernas, necesitamos comunicarnos con servidores o APIs externas para obtener datos o guardar información. Esta es la base de lo que se llama persistencia: guardar información de manera que no se pierda cuando la aplicación se cierre o recargue. En Angular, podemos usar el servicio HttpClient para manejar esta comunicación con el servidor y la persistencia de datos. Para ello, se utilizan los métodos GET y POST (además de otros como PUT, DELETE).

Una API (Interfaz de Programación de Aplicaciones, por sus siglas en inglés) es un conjunto de reglas y herramientas que permiten que diferentes aplicaciones o sistemas se comuniquen entre sí. Es como un puente que permite que dos aplicaciones intercambien información de manera estructurada, sin necesidad de que una conozca los detalles internos de la otra.

Para crear este servicio vamos a utilizar una API pública huachitos.cl esta Api nos proporciona información sobre perritos, como nombre, edad, personalidad y una imagen.

Lo inicial es crear el servicio en nuestra app de angular, creamos una carpeta llamada services y dentro de ella creamos un archivo llamado fake-store.service.ts este servicio se crea con el comando ng generate service fake-store.service

Asi debería verse hasta ahora:

Según la documentación de la API, la url de la api es https://huachitos.cl/api/animales/tipo/perro desde esa ruta vamos a consumir la información de los perros.

En el servicio que acabamos de crear, generamos una funcion que es la que se va a encargar de consumir la información de la api.

Ahora lo que vamos a hacer es llamar este servicio desde nuestro componente.

En este ejemplo se creó un componente con el nombre service-component, ya que se supone que es una página de adopción.

Como puedes observar, se importa el servicio, se inyecta en el constructor, se crea una función para llamar el servicio y guardarlo en la variable dogs. Este servicio se llama en el ngOnInit para que lo haga cuando se inicie el componente.
